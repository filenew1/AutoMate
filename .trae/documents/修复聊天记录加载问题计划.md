# 聊天记录不显示问题分析与修复计划

## 问题描述
用户与智能体对话后刷新页面，聊天记录为空。按照设计，聊天记录应该保存到 IndexedDB 中，并在下次打开时自动加载显示。

## 问题分析

### 1. 数据保存流程
检查 `ChatContainer.tsx` 中的保存逻辑：
- 用户发送消息 → 调用 `saveChatMessage()` 保存到 IndexedDB ✓
- AI 回复消息 → 调用 `saveChatMessage()` 保存到 IndexedDB ✓
- 技能激活 → 调用 `saveSkillCall()` 保存技能调用 ✓

### 2. 数据加载流程
检查 `ChatContainer.tsx` 中的加载逻辑：
```typescript
useEffect(() => {
  const loadHistory = async () => {
    if (!agentId) return
    try {
      const historyMessages = await getLast24HoursChatMessages(agentId)
      if (historyMessages.length > 0) {
        const existingMessages = chats[agentId]?.messages || []
        if (existingMessages.length === 0) {
          // 加载历史消息...
        }
      }
    } catch (err) {
      console.error('加载历史记录失败:', err)
    }
  }
  loadHistory()
}, [agentId])  // 问题：chats 不在依赖中
```

### 3. 可能的问题原因

**问题A: useEffect 依赖项不完整**
- `chats` 没有在 useEffect 的依赖数组中
- 虽然 `chats` 来自 zustand store 是响应式的，但显式声明依赖是最佳实践

**问题B: IndexedDB 查询逻辑问题**
- `getLast24HoursChatMessages` 使用复合索引查询
- 当前逻辑：`IDBKeyRange.bound([agentId, ''], [agentId, sinceStr])`
- 这可能不正确，因为复合索引的比较是字典序

### 步骤 2: 修复 IndexedDB 查询逻辑
文件：`src/services/chatHistoryService.ts`

修改 `getChatMessages` 函数，简化查询逻辑：
```typescript
export async function getChatMessages(
  agentId: string,
  since?: Date
): Promise<ChatMessageRecord[]> {
  const db = await getDB();
  let messages: ChatMessageRecord[];
  
  if (since) {
    // 先获取该智能体的所有消息，再在内存中过滤时间
    messages = await db.getAllFromIndex('chat_messages', 'by-agent', agentId);
    const sinceStr = since.toISOString();
    messages = messages.filter(m => m.send_time >= sinceStr);
  } else {
    messages = await db.getAllFromIndex('chat_messages', 'by-agent', agentId);
  }
  
  return messages.sort((a, b) => 
    new Date(a.send_time).getTime() - new Date(b.send_time).getTime()
  );
}
```
